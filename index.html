What it does

One sentence: point the phone at a shelf, and instantly see which products are in the wrong place.

The problem

Retail merchandisers check hundreds of shelves per day. Today they compare a printed planogram against what they see by eye — slow, inconsistent, and hard to audit.

A phone that can do this in real time turns a 20-minute manual check into a 20-second scan.

The approach

A hybrid multi-stage pipeline: one model finds product-shaped things (trained on 1.7M boxes from SKU110K), a second identifies the brand (34-class energy-drink classifier), and app logic handles shelf grouping + compliance.

Stages are independent, so adding a new SKU doesn't require retraining the generic detector.

Pipeline

Each stage is testable, swappable, and independently trained.

Stage 1

Generic detector

YOLOv8s trained on SKU110K (1 class). Finds every product-shaped thing on the shelf, regardless of brand.

Stage 2

Brand classifier

34-class YOLOv8 trained on energys. Falls back to MobileNetV3 embeddings in Phase 3.5.

Stage 3

Shelf grouping

Pure-Dart greedy clusterer: groups detected boxes by y-coordinate into rows.

Stage 4

Compliance overlay

Compares detected rows vs planogram.json. CustomPainter draws green/red boxes on the live preview.

Roadmap

Phased delivery — clear handoffs between scaffold, inference, compliance, and active learning.

  • Done Phase 1 — Project scaffold

    Flutter project, GitHub Actions CI, .gitignore, planogram schema.

  • Done Phase 2 — Core inference scaffold

    CameraService, Dart Isolate + TFLite runtime, Android config, compliance-engine stubs.

  • Next Phase 3 — Full compliance engine

    Shelf grouping + planogram comparison + CustomPainter overlay. First demo-ready build.

  • Planned Phase 3.5 — Embedding knowledge base

    MobileNetV3 embeddings, nearest-neighbour fallback, auto-save low-confidence crops, DBSCAN on unknowns.

  • Planned Phase 4 — Review UI

    In-app review of pending crops → swipe-to-label → bulk push back to Roboflow.

  • Speculative Phase 5 — Multi-category expansion

    Generalise beyond energy drinks: sparkling water, juices, etc. Stage 1 unchanged; new Stage 2 classifier per category.

Key decisions

A running log — each with the reason and consequences. Full log in docs/decisions-log.md.

2026-04

SKU110K stays separate from energys

Merging would dilute a 1.7M-box generic detector with 129 brand images. Separation of concerns wins.

2026-04

FP16 export, not INT8

INT8 calibration needs data.yaml that wasn't available. Revisit later.

2026-04

JPEG uploads to Roboflow

WebP caused annotation-matching failures. YOLOv8 rescales internally, so format is irrelevant to quality.

2026-04

Colab bootstrap for Label Assist

Roboflow Free plan blocks workspace-model Label Assist. External Colab loop is the workaround.

2026-04

50/30/20 split for small datasets

Default split gave 3 val images. One FN tanks mAP by 33%. Bigger val set = trustworthy numbers.

2026-04

Canonical class names

Merging adrenaline_250ml + adrenaline_original_250ml → single canonical. Full class review underway.
